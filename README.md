# R7's Online Guide — Online Privacy & Security

A practical, plain-language guide that gives everyday people the tools to be
safer and more private online. No jargon, no products to buy from us, no
tracking.

*(replace with your real URL once GitHub Pages is turned on — see below)*

---

## What this is

The entire guide is a **single, self-contained HTML file** (`index.html`).
All of its styling, behaviour, and images live inside that one file. There is:

- **No tracking** — no analytics, no cookies, no fingerprinting.
- **No ads** — and no affiliate links; recommendations are the author's own
  opinion.
- **Nothing hidden** — anyone can read the whole thing with their browser's
  **View Source**. That is the point.

Because it is one plain file, it works offline, loads instantly, can be saved
or emailed as-is, and can be hosted anywhere.

## Why it lives on GitHub

Hosting the guide in a public repository makes every change **transparent and
verifiable**:

- The **commit history** is an independent, tamper-evident record of what
  changed and when — separate from what the page itself claims.
- The **[CHANGELOG.md](CHANGELOG.md)** and the in-page *"What's New"* panel
  tell readers, in plain English, what changed in each version.
- **Releases / tags** (`v1.7`, `v1.6`, …) let anyone view or download the
  exact guide as it stood at any point in time.

This is the open-source spirit applied to a safety guide: you shouldn't have
to *trust* the advice blindly — you can inspect its whole history.

## How to read it

Just open the live site, or download `index.html` and open it in any browser.
Start with the **"Start here — the first five steps"** panel at the top, and
use the **Plain-English Glossary** at the bottom for any unfamiliar term.

## Contributing / corrections

Spotted a dead link, an out-of-date recommendation, or a mistake? Two ways to
help:

1. **Open an Issue** describing what's wrong (easiest — no coding needed).
2. **Open a Pull Request** with the fix, if you're comfortable editing HTML.

Privacy and security move fast, so link rot and stale advice are the most
common problems. Corrections are very welcome.
